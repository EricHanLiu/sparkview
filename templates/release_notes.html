SparkView Release Notes

This page serves as an archive for release notes and information about each version. The green version is currently deployed.


June 28th, 2019 v1.1.0
Github link
  • New backup system allows backups to add hours to accounts, edit budgets
  • Backups get allocated hours from the member that they're backing up
  • New Budgets (BETA) feature, robust budget monitoring (can exclude campaigns, flexible options)
  • Refactored the way that spend is pulled into the API (via campaigns now instead of account level)

May 23rd, 2019 v1.0.7
Github link
  • Created a warning system to alert some admin members if cron jobs are failing
  • Lists of members in the member and client areas will no longer include former members
  • Some admin reports will now have summary rows
  • Some new bloomer photos

May 14th, 2019 v1.0.6
Github link
  • Now storing historical management fees in the Account History report
  • Can now "complete" a mandate
  • Nominator is now in the High Five report
  • Add mandate hours to actual hours

May 6th, 2019 v1.0.5
Github link
  • Quick-add hour inputting on client page (clock icon)
  • Input hours fully moved to profile page, no longer in client dropdown
  • End of day summary popup at 4PM
  • Quick-add hours (15m, 30m, 1h) from profile input_hours page

May 1st, 2019 v1.0.4
Github link
  • Made updates to Oops and High Five reports
  • Can now add campaign groups that only have negative keywords
  • Can now input hours from the member profile
  • Can now add actual hours to a mandate assignment
  • Fixed bug that was breaking some member hour history records

April 24th, 2019 v1.0.3
Github link
  • Can now add "Additional services" to clients (services that have a start and end date)
  • Can make high-five and incident reports in SparkView now
  • Serious simplification of the main navigation menu
  • Removed some tabs on the member page

April 16th, 2019 v1.0.2
Github link
  • Can now view historical data in Team Lead Dashboards
  • Actual hours, available hours, and allocated hours will now be stored monthly per member
  • Several bug fixes of the team lead dashboard
  • Fix onboarding checklist

April 8th, 2019 v1.0.1
Github link
  • Added team leads dashboard: grouped reports filterable by team and role, data charts
  • Added project yesterday spend to budgets
  • Added management fee to account history report
  • Added opportunity and pitched reasons to the sales report

April 3rd, 2019 v1.0.0
Github link
  • Added services concept: can now flag opportunities and log sales pitches
  • Added negative keywords to text based campaign groups
  • Added sales report (outlines every service's status of every account)
  • Change in service status will now be logged and stored in the database
  • Various bug fixes
  • This is the base version. Everything that has been made in SparkView up until this point.
